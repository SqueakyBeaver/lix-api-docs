Detailed Description

A generation is a revision of a profile.

Each generation is a mapping (key-value pair) from an identifier (number) to a store object (specified by path).

◆ number

GenerationNumber nix::Generation::number

The number of a generation is its unique identifier within the profile.

◆ path

Path nix::Generation::path

The store path identifies the store object that is the contents of the generation.

These store paths / objects are not unique to the generation within a profile. Lix tries to ensure successive generations have distinct contents to avoid bloat, but nothing stops two non-adjacent generations from having the same contents.
